## About the project

Automation workflows for WRProjects Open Source Program Office.

## Self-Service

WRProjects developers who wish to fork an open source repository into [`/wr-projects`](https://github.com/wr-projects/) for upstream contributions can now leverage our self-service PR workflow.

> ðŸŽ‰ Please review the **[Fork a repository WIKI](https://github.com/wr-projects/ospo/wiki/fork-a-repository)** to get started!

## Internal Usage 

Currently, the OSPO Automation project has a unique workflow for creating repository forks. Note that currently this workflow is only for internal use by the WRProjects team.
